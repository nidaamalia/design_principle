Full Drop Zone

The primary upload variant. Use when file upload is the main action of a step, page, or form section.

Default multiple · accept=all · maxSizeMB=10
ⓘ The drop zone stays active while files are uploading — users can queue additional files without waiting for the current upload to finish.
⚠ When multiple=false, disable the drop zone immediately on upload start. Re-enable it only after success or error — never allow double-upload.

Inline (Form Field Variant)

Use when a file upload is one of several fields in a form row — same visual weight as a text input or date picker. Does not show a drop zone; the entire field acts as a click trigger.

Props API

All configurable props — applies to both the React component and the vanilla JS implementation.

Prop Type Default Description
File Constraints
accept string[] see → Accepted file types. Mix of MIME types and extensions:
['application/pdf','image/*','.xlsx','.csv']. Extensions starting with '.' are matched against the filename; MIME types and wildcards against the browser's file type.
maxSizeMB number 10 Maximum file size in megabytes. Files exceeding this are rejected client-side before any network call.
multiple boolean true Allow multiple file selection. Set false for single-file contexts. When false, the drop zone disables during upload and re-enables on completion.
maxFiles number 10 Maximum number of files per session. Only applies when multiple=true. Files beyond this limit are rejected with an inline message.
Variant & Layout
variant 'full'
'inline'		 'full' 'full' — large drop zone (default). 'inline' — form-field height trigger, no drop zone, single file only.
showFileList boolean true Show the attached files list below the drop zone. Set false if you manage the file list externally.
Behavior
autoAdvance boolean false If true, skip the success state entirely — call onAllSuccess immediately after upload completes. No "Next" button required. Used in wizard flows where the upload step auto-advances.
disabled boolean false Disable the entire component — zone, button, and file interactions are all non-interactive.
required boolean false Marks the field as required. Shows asterisk on label. Triggers error state when form is submitted without a file.
Content
label string 'Drag & drop…' Main headline inside the drop zone. For inline variant, this is the field label above the trigger.
placeholder string 'Choose file…' Placeholder text inside the inline trigger when empty.
hint string undefined Optional helper text shown below the limits line inside the drop zone. Shown in accent color. Use for contextual notes like "Selecting a saved template skips the next step."
initialFiles FileEntry[] [] Pre-populate the file list from server data. Each entry: { id, name, size, type, uploadedBy?, uploadedAt?, url? }. These render with full metadata (uploader, date) and action buttons.
Callbacks
uploadFn async function undefined async (file, onProgress) => response. Your upload implementation. Call onProgress(0–100) to update the progress bar. If omitted, the component simulates upload (demo mode).
onSuccess function undefined (file, response) => void. Called after each individual file upload completes.
onAllSuccess function undefined (files) => void. Called when all queued files finish uploading. Use with autoAdvance=true for wizard auto-progression.
onError function undefined (file, error) => void. Called when an upload fails (network error, timeout, server error). Does not fire for client-side validation rejections.
onCancel function undefined (file) => void. Called when user cancels an in-progress upload. The component resets to idle without confirmation.
onRemove function undefined (fileEntry) => void. Called after the user confirms removal of an already-uploaded file. Requires confirmation dialog for persisted files; no confirmation needed for in-progress cancels.

Usage Examples

Common configurations. Copy, adjust props, done.

Multi-file (general purpose)

// General multi-file drop zone — inspection reports, evidence photos, documents <FileUploadZone accept={['.pdf', 'image/*', '.xlsx', '.docx']} maxSizeMB={10} multiple={true} maxFiles={10} uploadFn={myUploadFn} onSuccess={(file, res) => handleSuccess(file, res)} onRemove={(entry) => handleRemove(entry)} />

Single-file, wizard auto-advance

// Wizard step — single file, auto-advances on upload success (no "Next" button needed) <FileUploadZone accept={['.xlsx', '.xls', '.csv']} multiple={false} autoAdvance={true} label="Drag & Drop your spreadsheet here" hint="Selecting a saved template skips the column mapping step." uploadFn={uploadSpreadsheet} onAllSuccess={() => goToStep2()} />

Inline — single field in a form row

// Inline variant — lives inside a form row alongside text/date fields <FileUploadZone variant="inline" accept={['.pdf', '.jpg', '.png']} label="Attachment" placeholder="Choose file…" required={false} uploadFn={uploadAttachment} onSuccess={(file, res) => setAttachmentId(res.id)} />

Pre-populated from server

// Load existing attachments from server + allow adding more <FileUploadZone initialFiles={serverFiles} // [{ id, name, size, type, uploadedBy, uploadedAt, url }] multiple={true} uploadFn={uploadFn} onRemove={async (entry) => { await api.deleteFile(entry.id); }} />

Rules & Guidelines

Behavior rules that apply across all variants.

ⓘ Always show accepted types and size limits inside the zone — not only in a tooltip or documentation. Users need this before searching for the file.
ⓘ Drag-over must give clear visual feedback — border turns solid accent color, background tints. Without this, users cannot tell if the drop target is active.
ⓘ Error rows persist until the user resolves them. Never auto-dismiss failed uploads. The user needs to decide whether to retry or remove.
ⓘ File name is always a link — clicking opens preview (PDF, image) or triggers download (XLSX, DOCX). Use the same accent color as other navigation links.
⚠ Validate type and size client-side immediately — before any server call. Rejected files never enter the uploading state.
⚠ Errors use inline messaging, never toasts. Toast notifications are easy to miss. File errors must be persistent and adjacent to the trigger element.
⚠ Removing an already-saved file requires confirmation. Canceling an in-progress upload does not — it hasn't been persisted yet.
⚠ The inline variant is single-file only. For multiple files in a form context, render a full drop zone section below the form row rather than cramming multi-file into an inline trigger.
! Handle edge cases explicitly: multiple files dropped in single-file mode (accept first, notice for rest), folder dropped (reject immediately), timeout (treat as error), network drop mid-upload (show error, re-enable).
